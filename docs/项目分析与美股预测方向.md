# Trade 项目分析:原理、美股方向、可行性与优化路线

> 分析日期:2026-07-03。基于当前 master 分支代码与 docs/ 计划文档的通读。

---

## 一、如何快速上手理解这个项目

### 1.1 一句话概括

这是一个 **"新闻/事件驱动 + 技术面因子"的 A 股每日选股推荐系统**,核心思想是 EBRT 架构:
**Evidence(证据)→ Belief(信念)→ Recommendation(推荐)→ Trust(信任度校验)**。

它不是高频交易系统,也不是端到端深度学习模型,而是一条**可解释、可审计的贝叶斯式信念更新流水线**:每天摄入新闻,抽取事件,通过知识图谱传播到相关股票,更新每只股票的"信念值",再结合技术指标打分排序,输出 buy / watch / avoid 建议,并用事后 Brier 分数等指标持续校验自己"值不值得信"。

### 1.2 技术栈分层

| 层 | 技术 | 位置 | 职责 |
|---|---|---|---|
| 计算内核 | C++ (clang-13 + CMake) | `engine/` | 高性能特征计算、回测、LightGBM/XGBoost 训练、Parquet/DuckDB 存储 |
| 业务主体 | Python (uv 管理) | `trade_py/` | 数据抓取、事件抽取、信念更新、推荐决策、评估 |
| Web UI | FastAPI + 前端 | `trade_web/` | Today 页、个股解释、K 线叠加信念曲线、运维工作台 |
| 统一入口 | `./trade` 脚本 | 根目录 | `./trade py ...`(Python CLI)、`./trade cli ...`(C++ CLI)、`./trade web` |
| 存储 | DuckDB + Parquet | `data/.db/trade.db`、`data/market/kline/` | 46 张表,K 线按股票存 Parquet |

### 1.3 核心数据流(读懂系统的主线)

```
新闻源(RSS/RSSHub/GDELT)
  → 文章摄入 (trade_py/data/news/)
  → LLM/NLP 富化:事件抽取、情绪打分 (trade_py/intelligence/, clients 支持 Anthropic/Ollama)
  → 知识图谱传播:事件 → 相关股票,带 hop 衰减 (trade_py/intelligence/graph/)
  → 证据规范化 Evidence (trade_py/evidence/)
  → 信念更新 BeliefState (trade_py/belief/)     ← 系统的"心脏"
  → 三因子排序打分 (trade_py/decision/rank.py)
  → 推荐 Recommendation + 全链路溯源 Trace
  → 事后评估:Brier 分数、漂移检测、Trust Gate (trade_py/evaluation/)
```

### 1.4 三个必读的核心公式(都在代码里)

**① 注意力加权**([belief/attention.py](../trade_py/belief/attention.py)):每条证据按"方向相似度 + 幅度 + 可靠性 + 新颖度 − 噪声 − 与当前信念的冲突"算 logit,softmax 得到权重——决定哪条新闻更值得信。

**② 残差信念更新**([belief/update.py](../trade_py/belief/update.py)):

```
b_{t+1} = (1−λ)·b_t + η·Σ wᵢ·Δᵢ        # λ=0.1 每日衰减, η=0.4 基础学习率
η 会被 TrustGate、漂移度、信源可靠性动态缩放
```

维护 mu_1d / mu_5d / mu_20d 三个时间尺度的信念(短期更敏感、长期更平滑)。

**③ 三因子推荐打分**([decision/rank.py](../trade_py/decision/rank.py)):

```
Score = (0.4×belief_mu + 0.3×window_score + 0.3×event_kg_score) × (1 − risk×0.5)
action: score>0.65 且 σ<0.25 → buy;  risk>0.6 → avoid;  其余 → watch
```

注意:**这些权重全是手写的启发式常数,不是学出来的**——这是理解系统现状的关键(见第三节)。

### 1.5 上手路径建议(按顺序)

1. **读文档**:`docs/05_plan_EBRT_01.md`(EBRT 总纲,含全部 10 张核心表和公式)→ `docs/21_plan_EBRT_17_causal_decision_architecture.md`(最新的因果决策链)。docs/ 下 01→22 按时间排列,就是项目的演进史。
2. **跑起来**:`./trade doctor` → `./trade setup-python` → `./trade web`,浏览器看 http://localhost:8080 的 Today 页。
3. **喂数据**:`./trade py data kline sync --mode range --symbols 600000.SH --start 2024-01-01 --end 2024-01-31`,再跑 `./trade py data sentiment --dry-run` 看新闻管道。
4. **读代码主线**:`trade_py/engine/__init__.py`(唯一 API 入口,薄层)→ `belief/` → `decision/` → `evaluation/gate.py`。
5. **看因子清单**:`trade_py/factors/definitions.py`,32 个特征列(事件类 + 情绪类 bf_* + 技术类 tech_*),一目了然系统"看什么"。

## 二、如果想预测美股,有什么方向

### 2.1 先说现状:系统目前是 A 股专用的

- K 线数据源只有 akshare / tushare / baostock([trade_py/data/market/kline/](../trade_py/data/market/kline/)),providers.py 里硬编码了 `ensure_a_share_symbol`、`.SH/.SZ` 后缀推断。
- 新闻源是新浪、财联社、格隆汇、东方财富、华尔街日报中文版等中文源。
- 涨跌停(price_limit)、换手率等特征假设也是 A 股规则。

### 2.2 改造成美股的工程路径(架构上是通的)

好消息是 EBRT 架构本身**市场无关**,需要换的是数据适配层:

1. **K 线 Provider**:新增 `yfinance`(免费,日线够用)或 `Polygon.io` / `Alpaca`(更可靠,有免费档)provider,实现现有的 `KlineProvider` Protocol(`fetch(symbol, start, end, adjust)`)即可插入。符号规范从 `600000.SH` 换成 `AAPL` 这类,需要把 `a_share_symbols` 工具抽象成 per-market 策略。
2. **新闻/事件源**:GDELT 通道已存在(`data/news/gdelt/`),天然覆盖英文全球新闻;再加 SEC EDGAR(8-K/10-Q 文件,免费 API)、财报日历、Fed 事件日历。美股的**信息披露远比 A 股规范**,8-K 事件类型本身就是结构化的事件标签——对这套事件驱动架构反而是利好。
3. **知识图谱**:公司—行业—供应链关系需要重建(可用 GICS 行业分类 + 13F 持仓 + 财报提及共现)。
4. **交易规则层**:去掉涨跌停逻辑,加上盘前盘后、除权除息(yfinance 自带复权)。

### 2.3 美股上"什么信号可能有效"——方向建议(按可行性排序)

1. **财报事件驱动(最契合现有架构)**:PEAD(盈余公告后漂移)是美股文献中最持久的异象之一——财报超预期后股价会持续漂移数周。现有的 Evidence→Belief 管道几乎就是为这个设计的:8-K/财报作为高可靠性证据,belief 的 mu_5d/mu_20d 多时间尺度正好匹配漂移周期。
2. **新闻情绪 + 中小盘**:大盘股(AAPL 等)的新闻在几分钟内被定价,日频系统没有优势;**中小盘、分析师覆盖少的股票**新闻反应慢,日频情绪信号仍有文献支持的预测力。
3. **横截面动量/质量因子**:tech_* 因子直接可复用,美股上 12-1 动量、低波动、质量因子有几十年的学术验证,作为 baseline 和风控层。
4. **宏观 regime 切换**:C++ 引擎里已有 `regime_detector`,美股对 Fed 利率周期、VIX regime 非常敏感,cross_asset 信号(已有 gold/fx/btc 通道)可以扩展成 regime 门控。
5. **不建议的方向**:日内/高频(基础设施不匹配)、单纯用 LLM 直接预测涨跌(噪声大且不可审计,违背本项目的 Trust 哲学)。

## 三、可行性评估:预测效果有多好?

### 3.1 直接结论

**目前无法证明有任何预测效果——因为系统还没有产生过可评估的记录。**

我检查了本地数据库 `data/.db/trade.db`:46 张表结构齐全,但 `Recommendation`、`BeliefState`、`Evidence`、`QualityReport`、`signals`、`causal_validation_outcomes` **全部是 0 行**。K 线数据只有一只股票(600000.SH)。也就是说:

- 评估框架(Brier 分数、Trust Gate、因果验证)**设计得很好,但从未在足量真实数据上运转过**;
- docs/21 号文档自己也明确承认:因果链目前是 "heuristic causal scaffold,不是已校准因果模型","expected_return 是保守启发式推断,明确打上 calibrated=false"。

### 3.2 架构层面的诚实评价

**做得好的地方(高于业余项目的平均水平):**

- 全链路溯源(RecommendationTrace)、事后打分(Brier)、漂移检测(MMD)、信源可靠性动态更新——**"系统知道自己什么时候不可信"** 这个设计比大多数量化爱好者项目成熟。
- 防泄漏意识:hfq 后复权、事件成熟窗口(matured window)评估、数据新鲜度追踪。
- 工程质量:分层清晰、迁移脚本、测试覆盖、C++/Python 职责分明。

**制约预测力的根本问题:**

1. **核心参数全是拍脑袋的**:0.4/0.3/0.3 的因子权重、λ=0.1、η=0.4、buy 阈值 0.65——没有一个是从数据中学习或校准的。
2. **没有真正的回测**:C++ 引擎有 backtest 模块,但 Python 侧 EBRT 链路(信念→推荐)没有历史回放验证,不知道这套打分历史上能不能跑赢基准。
3. **信号源天花板**:日频新闻情绪 + 常见技术指标,在有效市场里单独的信息比率(IR)都很低。中文财经新闻噪声大、信源可靠性参差,LLM 抽取事件还会引入额外噪声。
4. **样本量问题**:事件驱动信号天然稀疏,一只股票一年可能就几个有效事件,统计显著性需要大横截面 + 长时间积累。

**合理预期**:这类日频"新闻+技术面"系统,即使做得好,现实中的年化超额(after cost)通常在个位数百分点,信息比率 0.3~0.8 就算不错;当前未校准状态下,应假设**预测力约等于零,直到回测证明否则**。

### 3.3 如何优化,让预测更准(优先级排序)

**第 0 步(最重要):先建立评估基线,再谈优化。**
没有回测,一切"优化"都是盲调。具体做法:

1. **批量拉历史 K 线**(全市场或至少沪深300成分,3~5 年),回放式生成历史信号,用现有 Brier/命中率框架评估 T+1/T+5/T+20 的**信息系数(IC)**。给每个因子(belief_mu、window_score、kg_score、每个 tech_*)单独算 IC——先知道哪些因子有效、哪些是噪声。
2. **建立朴素基准**:随机选股、动量 top-N、等权指数。你的推荐必须稳定跑赢它们才有意义。

**第 1 层:把手写权重换成学习的(投入产出比最高)。**

- 用已有的 32 维因子([factors/definitions.py](../trade_py/factors/definitions.py))+ C++ 引擎里现成的 LightGBM,训练一个横截面排序模型(目标:未来 5 日超额收益的 rank),替代 rank.py 的线性加权。**必须用 purged walk-forward 交叉验证**(按时间切,留 embargo 期防泄漏)。
- belief 的 λ、η、注意力公式的 α/β/γ 等超参,用回测目标函数(如 rank IC)做贝叶斯优化。

**第 2 层:提升信号质量。**

- **信源分层**:利用已有的 InfluenceSignal/可靠性机制,大幅降权低质量源;交叉源确认(bf_cross_source_confirmation)作为硬过滤而非软特征。
- **事件标签校准**:对 LLM 抽取的事件类型,统计每类事件的历史平均后续收益和方差,把"事件→方向/幅度"从模板猜测变成经验分布(现有 event_templates 表正好可以放)。
- **加入基本面因子**:估值(PE/PB 分位)、盈利修正、财报意外——C++ 引擎已有 fundamental_features 骨架。

**第 3 层:组合与风控层面提效(往往比选股本身更能改善净值曲线)。**

- 行业/风格中性化:去掉推荐名单里的行业集中度,收益更多来自真 alpha 而非押行业。
- 波动率目标 + 已有的 Kelly/position_sizer 做仓位,而不是等权。
- Regime 门控:高波动/系统性风险期自动降杠杆(regime_detector 已有)。

**第 4 层(做美股时同步做):**

- 用 PEAD/财报事件做第一个可验证的策略切片——事件定义客观、文献基准清晰,最容易证明"系统到底行不行"。

### 3.4 一个务实的里程碑建议

1. **M1(2~4 周)**:全市场 K 线回补 + 纯技术因子 LightGBM 排序模型 + walk-forward 回测报告(IC、分层收益、换手率、费后收益)。这一步不依赖新闻管道,最快能拿到"系统有没有预测力"的第一个真实答案。
2. **M2**:接入新闻/事件因子,对比 M1 增量 IC——量化"事件驱动到底加了多少分"。
3. **M3**:paper trading 3 个月,Trust Gate 真实运转,对比回测与实盘偏差。
4. **M4**:再决定是否扩展美股。
